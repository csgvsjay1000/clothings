## 3.2 日志文件

### 3.2.1 错误日志

通过 **show variables like 'log_error'** 可以定位到错误路径，然后进行查看。

### 3.2.2 慢查询日志

慢查询日志可以帮助DBA定位到可能存在问题的SQL语句。DBA应该每天或每隔几天对其进行检查，当查询语句超过设定时间就会记录到慢查询日志文件里。

若查询语句没有用到索引，且也打开了监控没有用到索引的查询，也会将该类语句记录到慢查询日志文件中。
